**HF CDN Bypass Implementation**
================================

**Task:** Implement the HF CDN Bypass to avoid Hugging Face API rate limits during data ingestion.

**Highest-Value Incremental Improvement:** Pre-list file paths once, embed in training script.

**Implementation Plan:**

1. **Single API Call to List File Paths:**
   - Create a script to list file paths for a specific date folder using `list_repo_tree(path, recursive=False)` from the Hugging Face API.
   - Save the list of file paths to a JSON file.

2. **Embed File List in Training Script:**
   - Modify the training script to load the pre-listed file paths from the JSON file instead of making multiple API calls to `list_repo_files`.
   - Use the pre-listed file paths to download the files directly from the CDN using `https://huggingface.co/datasets/{repo}/resolve/main/{path}`.

**Code Snippet:**

```bash
# List file paths for a specific date folder
#!/bin/bash

repo="your-repo-name"
date="2022-01-01"
file_paths=$(curl -s -X GET \
  https://huggingface.co/api/v1/repos/${repo}/tree?path=${date} \
  -H 'Authorization: Bearer YOUR_HF_TOKEN' \
  -H 'Content-Type: application/json')

# Save file paths to a JSON file
echo "$file_paths" > file_paths.json
```

```python

with open('file_paths.json', 'r') as f:
```

**Benefits:**

* Avoids Hugging Face API rate limits during data ingestion.
* Reduces the number of API calls, making the process faster and more efficient.
* Saves time and resources by reusing the pre-listed file paths.
