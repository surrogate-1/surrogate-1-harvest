# workio / ops

### High-Value Incremental Improvement for Workio Ops
#### Diagnosis
The Workio project requires enhancements in its operations to improve the overall system's reliability, performance, and efficiency. Based on the patterns and lessons learned, the highest-value incremental improvement that can be shipped in <2h is to implement a script to automate the process of restarting Lightning Studio when it stops due to idle timeout.

#### Implementation Plan
1. **Create a new script**: Create a new Bash script (e.g., `restart-lightning-studio.sh`) in the `/opt/axentx/workio` directory.
2. **Check studio status**: Use the Lightning API to check the status of the studio before each `.run()` call.
3. **Restart studio if stopped**: If the studio is stopped, restart it using the `target.start(machine=Machine.L40S)` method.
4. **Schedule the script**: Schedule the script to run every 10 minutes using `cron` to ensure the studio is restarted promptly.

#### Code Snippet
```bash
#!/usr/bin/env bash

# Check studio status
studio_status=$(lightning status)

# Restart studio if stopped
if [ "$studio_status" == "Stopped" ]; then
  lightning target start --machine L40S
fi
```
#### Example Cron Job
```bash
*/10 * * * * /opt/axentx/workio/restart-lightning-studio.sh
```
This improvement will help prevent training processes from dying due to idle timeouts and ensure that the Workio system remains operational and efficient.
