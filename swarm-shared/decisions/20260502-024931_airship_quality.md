#### Part 1: Optimizing Dataset Ingestion with HF CDN Bypass

**Problem Statement:** The current implementation of dataset ingestion is rate-limited by the HF API, causing delays and inefficiencies in the training pipeline.

**Solution:** Implement the HF CDN bypass pattern to download dataset files directly from the CDN, bypassing the API rate limit. This can be achieved by:

1. Pre-listing file paths once and embedding them in the training script.
2. Using the `https://huggingface.co/datasets/{repo}/resolve/main/{path}` URL to download files directly from the CDN.

**Code Snippet:**
```python
import json
import requests

# Pre-list file paths and save to JSON
file_paths = []
repo = "your-repo-name"
path = "your-path"
response = requests.get(f"https://huggingface.co/datasets/{repo}/tree/main/{path}")
file_paths = [file["path"] for file in response.json()["files"]]
with open("file_paths.json", "w") as f:
    json.dump(file_paths, f)

# Embed file paths in training script
with open("file_paths.json", "r") as f:
    file_paths = json.load(f)

# Download files directly from CDN
for file_path in file_paths:
    url = f"https://huggingface.co/datasets/{repo}/resolve/main/{file_path}"
    response = requests.get(url)
    with open(file_path, "wb") as f:
        f.write(response.content)
```
**Benefits:**

* Bypasses HF API rate limit, reducing delays and inefficiencies in the training pipeline.
* Improves dataset ingestion speed and reliability.
