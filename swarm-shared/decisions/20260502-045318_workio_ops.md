### High-Value Incremental Improvement for Workio Ops
#### Diagnosis
The Workio project requires enhancements in its operations to improve the overall system's functionality, efficiency, and user experience. Based on the patterns and lessons learned, the highest-value incremental improvement that can be shipped in <2h is to implement a script to automate the process of restarting Lightning Studio when it stops due to idle timeout.

#### Implementation Plan
1. **Create a new script**: Create a new Bash script (e.g., `restart-lightning-studio.sh`) in the `/opt/axentx/workio` directory.
2. **Add Bash shebang**: Ensure the script has a proper Bash shebang (`#!/usr/bin/env bash`) and make it executable (`chmod +x restart-lightning-studio.sh`).
3. **Implement restart logic**: Use the Lightning API to check the status of the studio before each run. If the studio is stopped, restart it using the `target.start(machine=Machine.L40S)` method.
4. **Schedule the script**: Add a cron job to run the script at regular intervals (e.g., every 15 minutes) to ensure the studio is restarted when necessary.

#### Code Snippet
```bash
#!/usr/bin/env bash

# Set Lightning API credentials and studio name
LIGHTNING_API_TOKEN="YOUR_API_TOKEN"
STUDIO_NAME="YOUR_STUDIO_NAME"

# Check studio status and restart if necessary
while true; do
  studio_status=$(lightning studio status --name $STUDIO_NAME)
  if [ "$studio_status" == "Stopped" ]; then
    lightning studio start --name $STUDIO_NAME --machine L40S
  fi
  sleep 900 # wait 15 minutes
done
```
#### Example Use Case
To use this script, simply add it to the cron table and ensure the Lightning API credentials and studio name are set correctly. The script will automatically restart the studio when it stops due to idle timeout, ensuring uninterrupted operation.

```bash
crontab -e
```
Add the following line to the cron table:
```bash
*/15 * * * * /opt/axentx/workio/restart-lightning-studio.sh
```
This will run the script every 15 minutes, ensuring the studio is restarted when necessary.
