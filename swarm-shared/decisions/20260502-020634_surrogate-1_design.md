# surrogate-1 / design

### Diagnosis
* The project lacks a robust data ingestion pipeline, which is crucial for training and updating the surrogate-1 model.
* The current implementation uses the Hugging Face API, which has rate limits that can block dataset training.
* The project does not have a mechanism to bypass the Hugging Face API rate limit, which can be achieved by using the HF CDN to download dataset files directly.
* The project's training script does not handle the case where the Lightning Studio is stopped due to idle timeout, which can kill the training process.
* The project's dataset mirror writes mixed-schema files to the enriched directory, which can cause issues with the training pipeline.

### Proposed change
The proposed change is to implement a robust data ingestion pipeline that uses the HF CDN to bypass the Hugging Face API rate limit. This will involve modifying the `train.py` script to download dataset files directly from the HF CDN using the `https://huggingface.co/datasets/{repo}/resolve/main/{path}` URL pattern.

### Implementation
To implement this change, we will need to make the following modifications:
```python
# train.py
import json
import os

# Load the list of dataset files from the JSON file
with open('dataset_files.json', 'r') as f:
    dataset_files = json.load(f)

# Download the dataset files from the HF CDN
for file in dataset_files:
    file_url = f"https://huggingface.co/datasets/{file['repo']}/resolve/main/{file['path']}"
    file_path = os.path.join('datasets', file['path'])
    os.makedirs(os.path.dirname(file_path), exist_ok=True)
    # Download the file using the requests library
    import requests
    response = requests.get(file_url)
    with open(file_path, 'wb') as f:
        f.write(response.content)
```
We will also need to create a `dataset_files.json` file that contains the list of dataset files to download. This file can be generated by running a script that uses the Hugging Face API to list the files in the dataset repository.
```python
# generate_dataset_files.py
import json
import os
import requests

# Set the repository name and path
repo_name = "your-repo-name"
path = "your-path"

# Get the list of files in the repository
response = requests.get(f"https://huggingface.co/api/v1/datasets/{repo_name}/tree?path={path}")
files = response.json()

# Save the list of files to a JSON file
with open('dataset_files.json', 'w') as f:
    json.dump(files, f)
```
