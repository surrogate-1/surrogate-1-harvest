### Implementing a Robust Data Ingestion Pipeline with HF CDN Bypass for Airship Ops
#### Problem Statement
Airship operations are currently hindered by Hugging Face API rate limits, which slow down data ingestion and hinder the overall performance of the platform.

#### Solution Overview
To address this issue, we will implement a robust data ingestion pipeline that leverages the Hugging Face CDN bypass. This approach enables us to download dataset files directly from the CDN, bypassing the API rate limits and ensuring uninterrupted data ingestion.

#### Implementation Plan

1. **Pre-list file paths**: Make a single API call to `list_repo_tree(path, recursive=False)` for one date folder and save the list of file paths to a JSON file.
2. **Embed file list in training script**: Embed the saved JSON file list in the training script (`train.py`) to enable direct CDN downloads.
3. **Use HF CDN bypass**: Modify the data ingestion pipeline to use the HF CDN bypass for downloading dataset files, ensuring that API rate limits are avoided.
4. **Implement retry mechanism**: Implement a retry mechanism to handle any potential errors or timeouts during data ingestion, ensuring that the pipeline remains robust and reliable.
