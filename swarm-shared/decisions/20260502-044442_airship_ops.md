### High-Value Incremental Improvement: Implementing HF CDN Bypass for Faster Data Ingestion
#### Problem Statement
The current data ingestion pipeline in Airship is limited by Hugging Face API rate limits, causing delays and affecting overall platform performance.

#### Solution Overview
To address this issue, we will implement the HF CDN Bypass strategy. This involves downloading public dataset files directly from the Hugging Face CDN, bypassing the API rate limit checks. We will pre-list file paths once, embed them in the training script, and use the CDN-only fetches with zero API calls during data load.

#### Implementation Plan
1. **Update `train.py`**: Modify the training script to use the HF CDN Bypass strategy.
2. **Pre-list file paths**: Make a single API call to `list_repo_tree(path, recursive=False)` for one date folder, and save the list to a JSON file.
3. **Embed file list in `train.py`**: Read the pre-listed file paths from the JSON file and use them to download dataset files directly from the Hugging Face CDN.
4. **Use CDN-only fetches**: Update the data loading process to use the CDN-only fetches with zero API calls.
