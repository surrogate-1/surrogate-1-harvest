Based on the provided patterns and lessons learned, the highest-value incremental improvement that can ship in <2h is to implement the HF CDN Bypass pattern to avoid rate-limit blocks during dataset training.

### Implementation Plan
1. **Identify the dataset**: Determine which dataset is being used for training and identify the corresponding repository on Hugging Face.
2. **Get the file list**: Use the `list_repo_tree` API to get a list of file paths for the dataset. This can be done using a single API call from the Mac.
3. **Save the file list to JSON**: Save the list of file paths to a JSON file that can be embedded in the training script.
4. **Modify the training script**: Update the training script to use the CDN-only fetches with zero API calls during data load.
5. **Test the implementation**: Test the implementation to ensure that it works as expected and does not encounter any rate-limit errors.

  -d '{"repo_id": "your_repo_id", "path": "your_path", "recursive": false}')

# Save the file list to JSON
echo "$file_list" > file_list.json
