# surrogate-1 / design

**Synthesized Solution**

**Diagnosis**
* The project lacks a robust implementation for handling Hugging Face API rate limits, which can block dataset training.
* There is inadequate reuse of existing Lightning Studio instances, leading to wasted quota and potential downtime.
* The project does not leverage the Hugging Face CDN for dataset downloads, which can bypass API rate limits.

**Proposed change**
* Implement a robust rate limit handling mechanism using the Hugging Face CDN for dataset downloads.

**Implementation**
1. Update the `design-cycle-2026-04-29.sh` script to download dataset files from the Hugging Face CDN using `list_repo_tree` and `hf_hub_download`:
```bash
#!/bin/bash

# ...

# Download dataset files from Hugging Face CDN
for file in $(huggingface_hub.list_repo_tree "your-repo-name", "path/to/folder", recursive=False); do
  hf_hub_download "your-repo-name" "$file"
done
```
2. Update the `train.py` script to use the downloaded files from the CDN instead of making API calls to `list_repo_files` and `download_file`. To achieve this, follow the implementation steps from Candidate 2:
```python
# train.py
import json
import os
import requests

# Load the list of file paths from the JSON file
with open('file_paths.json', 'r') as f:
    file_paths = json.load(f)

# Download the dataset files from the Hugging Face CDN
for file_path in file_paths:
    hf_hub_download(file_path)
```
3. Create a `file_paths.json` file containing the list of file paths to be downloaded from the Hugging Face CDN:
```json
[
    "https://huggingface.co/datasets/{repo}/resolve/main/{path}"
]
```

* The synthesized solution combines the strengths of both candidates, including the use of `list_repo_tree` and `hf_hub_download` to download dataset files from the Hugging Face CDN.
