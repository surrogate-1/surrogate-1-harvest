### Comprehensive Solution: Implementing HF API Rate Limit Fix and CDN Bypass

To address the HF API rate limit 429 error and improve the overall performance and reliability of the Costinel platform, we propose a comprehensive solution that combines the strongest insights from the provided proposals.

#### Problem Statement

The HF API rate limit 429 error occurs when the number of requests exceeds 1000 per 5 minutes, blocking dataset training and ingestion.

#### Solution

To fix this issue, we will implement the following solution:

1. **Avoid recursive `list_repo_files`**: Instead of using `list_repo_files` recursively on big repos, use `list_repo_tree(path, recursive=False)` per folder to reduce the number of API calls and avoid pagination.
2. **Use CDN downloads**: Use CDN downloads (`/resolve/main/`) instead of API calls to download dataset files, bypassing the API rate limit entirely.
3. **Wait 360s before retry**: If the 429 error occurs, wait 360 seconds before retrying the API call.
4. **Pre-list file paths**: Make a single API call to `list_repo_tree(path, recursive=False)` to get the list of file paths for the dataset and save it to a JSON file.
5. **Embed file list in `train.py`**: Read the JSON file containing the file list and use it to download the dataset files from the HF CDN.

#### Implementation Plan

1. Update the `granite-business-research.sh` script to use `list_repo_tree` instead of `list_repo_files`.
2. Modify the `train.py` script to use CDN downloads for dataset files and embed the file list.
3. Add a retry mechanism with a 360s wait time to the `train.py` script.
4. Test and verify the updated `train.py` script to ensure that it can download the dataset files from the HF CDN without hitting the rate limit.
